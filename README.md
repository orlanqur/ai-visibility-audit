# AI-Visibility Audit — система для Катерины

Эта система автоматизирует аудит видимости бренда в AI-моделях (Alice, ChatGPT, Google AIO, Perplexity, Yandex Neuro и т.п.). Ты получаешь на входе набор сырых файлов с ответами моделей, а на выходе — нарезку по вопросам, метрики, карту ссылок и готового ассистента в VSCode, который работает с этими данными без «переедания контекста».

---

## 0. Что внутри и зачем

**Проблема, которую решаем**: ты уже пробовала скармливать Claude файл одной модели (10–15к строк markdown). Он перечитывает его на каждое уточнение, переедает и начинает придумывать.

**Как лечим**:
1. **Режем** сырой файл одной модели на отдельные файлы по вопросам → каждый — компактный (5–100 строк) с YAML-заголовком.
2. **Индексируем** всё через MCP-сервер `docs-intelligence-mcp` (векторный поиск + метаданные). Claude ходит по нарезке точечно.
3. **Параллелим** разбор через skill `audit-split-by-model`: 5+ subagent-окон читают свои модели одновременно, головной Claude только синтезирует.

**Результат**: быстрый ответ без фантазий.

## 1. Что ставим на Mac (один раз)

Все шаги ниже — в **Терминале** (Cmd+Space → «Терминал» → Enter).

### 1.1. Homebrew — менеджер пакетов

Если у тебя его ещё нет, скопируй в терминал:
```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```
После установки терминал попросит добавить brew в PATH — следуй его подсказкам.

Проверка: `brew --version` должно вывести версию.

### 1.2. Python 3.11 и git

```bash
brew install python@3.11 git
```

Проверка:
```bash
python3.11 --version   # должно быть Python 3.11.x
git --version
```

### 1.3. VSCode

Скачай: https://code.visualstudio.com/ → «Download for Mac» → разархивируй → перенеси Visual Studio Code.app в «Программы».

### 1.4. Плагин Claude Code в VSCode

1. Запусти VSCode.
2. Слева в панели значок «квадратики» (Extensions), или Cmd+Shift+X.
3. В поиске набери **Claude Code** → нажми **Install**.
4. После установки в левой панели появится иконка Claude. Нажми её → кнопка «Sign in» → войди своим аккаунтом Anthropic (у тебя платная подписка — она работает). VSCode откроет браузер, подтвердишь авторизацию.

### 1.5. Клонируй репозиторий этой системы

В терминале:
```bash
cd ~
git clone <URL-репозитория-дам-я> ai-audit
cd ai-audit
```
(я пришлю тебе точный URL, когда зальём на GitHub)

### 1.6. Клонируй и установи docs-intelligence-mcp

Этот MCP-сервер даёт Claude семантический поиск по нарезке. Ставим его один раз:

```bash
cd ~
git clone <URL-docs-intelligence-mcp> docs-intelligence-mcp
cd docs-intelligence-mcp
python3.11 -m venv venv
source venv/bin/activate
pip install -e .
```

Проверка:
```bash
which docs-intel
# должно быть что-то вроде /Users/ты/docs-intelligence-mcp/venv/bin/docs-intel
```

### 1.7. OpenAI API-ключ

Этот ключ нужен MCP-серверу, чтобы превращать текст в векторы для поиска. **Ключ я дам тебе отдельно** — попроси у Аркадия в мессенджере.

Когда получишь — создай файл `.env` **внутри `~/ai-audit`**:
```bash
cd ~/ai-audit
nano .env
```
Вставь (замени `sk-...` на реальный ключ от меня):
```
OPENAI_API_KEY=sk-...
EMBEDDING_MODEL=text-embedding-3-small
```
Сохрани: Ctrl+O → Enter → Ctrl+X.

> Важно: файл `.env` в `.gitignore`, он никуда не уедет — ключ в безопасности.

## 2. Создаём первый аудит (каждый раз, когда новый бренд)

### 2.1. Заведи папку проекта

```bash
cd ~/ai-audit
mkdir -p projects/<имя_бренда_латиницей>/responses
```
Пример: `mkdir -p projects/alfa_bank/responses` или `mkdir -p projects/tinkoff/responses`.

### 2.2. Положи сырые файлы

В `projects/<бренд>/responses/` кидай файлы с ответами моделей.

**Имя файла строго**: `{Model}-{Brand}.md`
Примеры: `Alice-Tinkoff.md`, `ChatGPT-Tinkoff.md`, `Perplexity-Tinkoff.md`.

**Формат внутри** (важно — скрипт режет по этим маркерам):
```
# Brand / Model

### Вопрос 1
текст вопроса одной строкой

### Ответ 1
тело ответа (markdown, может быть длинным)

### Источники 1
1. https://...
2. https://...

---

### Вопрос 2
...
```

Если формат в следующем аудите чуть другой — правь regex в начале `scripts/slice.py` (3 константы наверху файла).

### 2.3. Открой проект в VSCode

```bash
cd ~/ai-audit
code .
```
Откроется VSCode с проектом.

### 2.4. Запусти Claude в плагине

Слева нажми иконку Claude → откроется чат. Первым сообщением напиши:

> «Привет. Я первый раз работаю с этой системой. Прочитай README.md и CLAUDE.md, расскажи мне кратко что это, и запусти skill `pipeline-guide` — он подскажет следующий шаг.»

Claude прочитает документацию и позовёт skill `pipeline-guide`, который проверит состояние проекта (есть ли сырьё, прогнался ли slice, и т.д.) и подскажет одну конкретную команду на следующий шаг.

### 2.5. Прогон пайплайна (Claude ведёт тебя за руку)

Skill `pipeline-guide` даст тебе одну команду за раз:

1. Нарезка:
   ```bash
   python scripts/slice.py --project <бренд>
   ```
2. Метрики:
   ```bash
   python scripts/metrics.py --project <бренд> --brand "Альфа-Банк" \
     --alias "Альфа-банк" --alias "Alfa Bank" --alias "alfabank.ru"
   ```
   (в `--alias` — все варианты написания бренда и его домен; `--alias` можно повторять)
3. Карта ссылок:
   ```bash
   python scripts/link_map.py --project <бренд> --min-models 3 --top 30
   ```
4. Индексация MCP (чтобы работал семантический поиск):
   ```bash
   cd projects/<бренд>
   source ~/docs-intelligence-mcp/venv/bin/activate
   docs-intel index .
   cd ../..
   ```

После этого система готова. Каждый раз спрашивай `pipeline-guide`, если растерялась — он видит состояние и подскажет.

## 3. Типовые запросы к Claude

Пиши ему прямо в чате VSCode:

### Слепые вопросы по одной модели
```
/blind-analysis Alice
```
Claude отфильтрует по `metrics.csv` слепые вопросы Alice, прочитает только нужные qNNN, соберёт отчёт с конкурентами-бенефициарами и зонами роста.

### Кросс-модельный срез по теме
```
/cross-model Мобильное приложение банка
```
Claude запустит skill `audit-split-by-model` → 5+ subagents параллельно разберут свои модели → головной агент синтезирует отчёт и сохранит в `projects/<бренд>/reports/`.

### Карта ссылок и консенсус
```
/link-map
```
Отчёт из `link_map.json` + `link_consensus.csv`: топ-агрегаторы, где бренд отсутствует, конкуренты в топе.

### Живой вопрос
```
Найди q-ID, где бренд не упомянут в источниках, но упомянут в теле ответа. По всем моделям. Сведи в таблицу.
```
Claude прочитает `metrics.csv` (не нарезку) и ответит.

### Запуталась
```
что мне сейчас делать?
```
Claude позовёт `pipeline-guide` и скажет один шаг.

## 4. Как работает «не переедает контекст»

Схема:
- **Ты** просишь: «разбери кросс-модельно тему X».
- **Главный Claude** зовёт skill → дёргает 5 subagents (по одному на модель).
- **Каждый subagent** у себя в чистом контексте читает только свою модель, возвращает 500-словный markdown-отчёт.
- **Главный Claude** получает 5 структурированных отчётов и синтезирует. Сырья не видит.

Пока держишься этой схемы через slash-команды и skill — Claude не «переедает».

## 5. FAQ / частые грабли

**`docs-intel: command not found`** — ты не активировала venv. Запусти `source ~/docs-intelligence-mcp/venv/bin/activate`. В каждом новом терминале — заново.

**`python scripts/slice.py --project ...` жалуется «нет .md»** — проверь, что файлы лежат именно в `projects/<бренд>/responses/`, а не в корне или в `responses/`.

**MCP не отвечает / Claude не видит `docs_search`** — проверь, что:
- в `~/ai-audit/.env` есть `OPENAI_API_KEY`
- `docs-intel index .` прогонялся в папке проекта
- VSCode перезапущен после редактирования `.env`

**Claude всё равно «фантазирует»** — ты, скорее всего, не через slash-команду/skill обращаешься. Попроси: «вызови skill audit-split-by-model».

**Формат сырых файлов поменялся (маркеры другие)** — правь regex в начале `scripts/slice.py`. Там три константы наверху.

**Как обновить систему, когда Аркадий что-то исправит** — в терминале:
```bash
cd ~/ai-audit
git pull
```
Твои `projects/` остаются на месте (они в `.gitignore`).

## 6. Когда захочешь расширить

- Добавить колонку `intent` (blind/branded, transactional/informational) — опциональным столбцом в `metrics.py` или отдельным LLM-шагом.
- SQLite-база аннотаций — если ручных пометок становится много. В MVP специально нет.
- Новая модель — просто клади `{Model}-{Brand}.md` в `responses/` и перезапускай пайплайн. Скрипты модель-агностичные.
- Несколько брендов одновременно — просто создавай `projects/tinkoff/`, `projects/sber/` рядом. Каждый — независимый.

## 7. Что делать если что-то непонятно

Открой Claude в VSCode и напиши обычным языком: «не могу понять что такое X», «у меня ошибка Y, вот лог». Claude знает про репо всё что нужно (см. `CLAUDE.md`), он тебе объяснит в контексте.

Если Claude сам не справился — пиши Аркадию.
