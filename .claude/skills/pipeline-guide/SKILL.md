# pipeline-guide

Помогает пользователю пройти пайплайн AI-аудита шаг за шагом. Проверяет текущее состояние проекта и говорит, какую команду запустить следующей. Используется в основном новичками (типа коллеги, которой ты передаёшь систему).

## Алгоритм

1. **Определи активный проект** по правилу из `CLAUDE.md`. Если проектов нет (`projects/` пуста или отсутствует) — подскажи создать: `mkdir -p projects/<имя>/responses` и положить туда сырые файлы.
2. Пройди чек-лист ниже **по порядку**. Первый же невыполненный пункт = следующий шаг для пользователя. Покажи ему конкретную команду и одну строчку «зачем».

## Чек-лист (выполняй glob/ls, не Read сырые файлы)

| № | Что проверить | Как проверить | Если не выполнено — следующий шаг |
|---|---|---|---|
| 1 | Есть ли сырые файлы? | `projects/<P>/responses/*.md` не пусто | «Положи сырые `{Model}-{Brand}.md` в `projects/<P>/responses/`. Формат см. README §3.2.» |
| 2 | Есть ли нарезка? | `projects/<P>/sliced/*/q*.md` не пусто И свежее `responses/` | `python scripts/slice.py --project <P>` — разрезает сырьё на 1 файл = 1 вопрос 1 модели |
| 3 | Есть ли метрики? | `projects/<P>/metrics/metrics.csv` существует И свежее `sliced/` | `python scripts/metrics.py --project <P> --brand "<Бренд>" --alias "..."` — посчитай упоминания бренда, объём ответов, позиции в источниках |
| 4 | Есть ли карта ссылок? | `projects/<P>/metrics/link_map.json` существует И свежее `sliced/` | `python scripts/link_map.py --project <P> --min-models 3` — построй инвентарь доменов и консенсус |
| 5 | MCP проиндексирован? | `projects/<P>/.docs-intel/` существует | `cd projects/<P> && docs-intel index .` — создаст векторный индекс, без него `docs_search` не работает |
| 6 | Всё готово | — | Подскажи 3–4 типовых стартовых запроса (см. ниже) |

## Если всё готово — что предложить

Выведи короткий список, чтобы Катерина выбрала:

- **«Разбери слепые вопросы по Alice»** → `/blind-analysis Alice`
- **«Построй карту ссылок»** → `/link-map`
- **«Сравни как модели отвечают про мобильное приложение»** → `/cross-model Мобильное приложение банка`
- **«Найди q-ID, где бренд не упомянут в источниках»** → «покажи строки `metrics.csv`, где `brand_in_sources=False`»

## Формат ответа

Коротко (≤200 слов). Одна команда на следующий шаг + 1 строка «зачем». Без пересказа всего README.

Если пользователь спросит «почему» или «что это значит» — тогда разворачивай подробнее, но сначала — один конкретный шаг.

## Антипаттерны

- Не запускай скрипты сам — покажи команду, пусть пользователь сам её запустит. Это обучение, не автоматизация.
- Не пересказывай README/CLAUDE.md — они уже есть. Ты — навигатор, не документация.
- Не выполняй несколько шагов сразу — один шаг, проверка, следующий шаг.
