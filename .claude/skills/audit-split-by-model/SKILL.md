# audit-split-by-model

Параллельный разбор AI-аудита по моделям. Цель — не давать головному агенту переедать контекст 10+к строк сырья: каждый subagent работает со своей моделью в чистом контексте, возвращает структурированный markdown ≤500 слов.

## Когда использовать

- «Разбери слепые транзактные вопросы по всем моделям»
- «Сравни, как модели отвечают на тему Х»
- «Собери кросс-модельный срез по бренду / конкуренту»
- Любая задача, где нужно обойти все модели и потом синтезировать

## Когда НЕ использовать

- Вопрос про одну модель → просто Read нужных файлов
- Вопрос, решаемый через `metrics.csv` / `link_map.json` → не запускай subagents ради агрегатов

## Алгоритм

1. **Определи активный проект** по правилу из `CLAUDE.md`. Обозначим `projects/<P>/`.
2. **Список моделей** = `ls projects/<P>/sliced/`.
3. **Сформулируй промпт-шаблон для subagent**. Подставь конкретную задачу пользователя. Обязательные поля в ответе subagent:
   - Ключевые наблюдения (3–7 пунктов)
   - Таблица q-ID → краткий тезис → ключевой источник
   - Цитаты (2–4) с q-ID
   - Упоминается ли бренд, в каком контексте
   - Конкуренты, цитируемые моделью по теме
   - **Ограничение длины ответа subagent — ≤500 слов.**
4. **Запусти subagents параллельно** — один Task call на модель, все в одном сообщении.
5. **Дождись всех результатов**.
6. **Синтезируй кросс-модельный отчёт**:
   - Общие паттерны (что модели говорят одинаково)
   - Расхождения (где спорят)
   - Консенсус-источники на тему
   - Где бренд упомянут, где нет
   - Рекомендации (3–5 пунктов)
7. **Сохрани** в `projects/<P>/reports/{{YYYY-MM-DD}}-{{topic-slug}}.md`.

## Шаблон промпта для subagent

```
Ты анализируешь ответы AI-модели {{MODEL}} для бренд-аудита.

Папка с нарезкой: projects/{{P}}/sliced/{{MODEL}}/ (файлы qNNN.md, YAML-frontmatter внутри)
Индекс метрик: projects/{{P}}/metrics/metrics.csv (отфильтруй по model={{MODEL}})
Бренд: {{BRAND}}

Задача пользователя: {{TASK}}

Алгоритм:
1. По metrics.csv отбери вопросы, релевантные задаче.
2. Прочитай только эти qNNN.md через Read (не весь sliced/).
3. Верни markdown ≤500 слов по структуре:
   ## Наблюдения (3-7 пунктов)
   ## Таблица q-ID | тезис | источник
   ## Цитаты (2-4, с q-ID)
   ## Бренд: упомянут? где? как?
   ## Конкуренты, которых модель цитирует

Не превышай 500 слов. Структурированный ответ ценнее подробного.
```

## Ключевое правило

**Головной агент НЕ перечитывает `sliced/`.** Все цитаты и тезисы — из отчётов subagent. Если нужна дословная цитата — Read только одного конкретного `projects/<P>/sliced/{model}/qNNN.md`, указанного в отчёте subagent. Это и есть решение проблемы «Claude переедает контекст».
