Задача: построить карту источников и отчёт по консенсусу.

Пред-шаги:

1. **Определи активный проект** (правило в `CLAUDE.md`). Обозначим путь `projects/<P>/`.

Алгоритм:

1. Если `projects/<P>/metrics/link_map.json` отсутствует или старше `projects/<P>/sliced/` — запусти:
   ```
   python scripts/link_map.py --project <P> --min-models 3 --top 30
   ```
2. Открой `projects/<P>/metrics/link_consensus.csv` — это домены у ≥3 моделей.
3. Опционально — сверься через `docs_url_inventory` из docs-intelligence-mcp.
4. Собери отчёт:
   - **Топ-10 консенсус-доменов** (число моделей, вопросов, упоминаний в источниках/теле).
   - **Наш бренд-домен** — сколько моделей его цитируют, в каких вопросах отсутствует. Найди q-ID через `metrics.csv` (`brand_in_sources=False`).
   - **Конкуренты в топе** — домены у ≥4 моделей, которые не являются бренд-доменом. Ранжируй по `sources_mentions`.
   - **Агрегаторы** (banki.ru, sravni.ru и т.п.) — отдельной секцией: у них своя роль, бренд там должен присутствовать.
5. Для секции «конкуренты» — выведи 3–5 конкретных URL конкурента с q-ID (поле `urls` в `link_map.json`).

Отчёт в markdown, таблицы допустимы. Не перечитывай `sliced/` — данные уже в JSON/CSV.
