---

Задача: собрать кросс-модельный срез по теме **{{topic}}**.

Пред-шаги:

1. **Определи активный проект** (правило в `CLAUDE.md`). Обозначим путь `projects/<P>/`.
2. Если `{{topic}}` не задана — спроси.

Алгоритм:

1. **Не читай `projects/<P>/sliced/` сам.** Вызови skill `audit-split-by-model` со следующим заданием для каждого subagent:
   > «Прочитай `projects/<P>/sliced/{model}/` и собери по теме `{{topic}}`: (а) какие вопросы (q-ID) её касаются, (б) как модель отвечает — ключевые тезисы, (в) какие источники использует, (г) упоминается ли наш бренд и в каком контексте. Формат — markdown ≤500 слов + таблица q-ID/ключевая мысль/источник.»
2. Дождись всех результатов.
3. Синтезируй **из собранных отчётов** (не из `sliced/`):
   - Что модели говорят одинаково, что — по-разному?
   - Где бренд упомянут, где — нет?
   - Кто «общие бенефициары» (цитируются в нескольких моделях на эту тему)?
4. Отчёт: сводная таблица по моделям, консенсус-тезисы, расхождения, топ-источники, рекомендации.
5. Сохрани в `projects/<P>/reports/{{YYYY-MM-DD}}-{{topic-slug}}.md`.

Правило: головной агент цитирует только то, что вернули subagents. Для точной цитаты — Read конкретного `projects/<P>/sliced/{model}/qNNN.md`, указанного в отчёте subagent.
