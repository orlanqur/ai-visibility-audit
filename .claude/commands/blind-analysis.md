---

Задача: разобрать слепые вопросы для модели **{{model}}**.

Пред-шаги:

1. **Определи активный проект** — правило из `CLAUDE.md`. Если в `projects/` одна подпапка, используй её; иначе спроси. Обозначим путь `projects/<P>/`.
2. Если `{{model}}` не задан — спроси (список моделей = `ls projects/<P>/sliced/`).

Алгоритм:

1. Открой `projects/<P>/metrics/metrics.csv` — это твой индекс. Отфильтруй строки `model={{model}}`.
2. «Слепыми» считай вопросы, в формулировке которых нет бренда. Текст вопроса — в колонке `question_text`. Бренд — из YAML-frontmatter любого sliced-файла (поле `brand`).
3. Раздели выборку на 2 группы:
   - **Бренд всё равно упомянут** (`brand_in_body=True`) — модель сама вытянула бренд. Хороший сигнал.
   - **Бренд не упомянут** (`brand_in_body=False`) — модель обошлась без нас. Зона роста.
4. Для каждой группы прочитай sliced-файлы через Read (не через `docs_search`). Собери:
   - Какие темы/интенты преобладают в зоне роста?
   - Кто **бенефициары** зоны роста — какие конкуренты цитируются вместо нас?
   - В группе «бренд всё равно упомянут» — какой контекст, какие формулировки?
5. Отчёт в markdown: сводка по группам, топ-3 конкурентов-бенефициаров, 5 примеров цитат из зоны роста со ссылкой на q-ID, 3–5 рекомендаций.

Правила:
- Цитаты — только дословно из sliced-файлов, с указанием `{{model}}/qNNN`.
- Не читай `projects/<P>/sliced/{{model}}/` подряд — только отфильтрованные по CSV q-ID.
- Если слепых вопросов > 15 — вызови skill `audit-split-by-model` для этой модели (не переедай контекст).
